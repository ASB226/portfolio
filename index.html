Project 1

국내 대기업 자동차 연구소 컨테이너 환경 구축 프로젝트

기간: 2023.09 - 2024.01

역할: 시스템 엔지니어

참여인원: 2명

기여도: 90%

AS-IS / TO-BE

AS-IS:연구원들이 관리 포털을 통해 Windows 379대(35%), Linux 658대(65%) 총 1,037대의 VM을 개별적으로 관리

TO-BE:기존 VM 환경 유지, 서버 5대 증설 및 Linux VM 중 140대(15%)를 컨테이너로 전환하여 컨테이너 관리 환경 구축

Architecture

VMware Tanzu ⚡CI/CD 📈Monitoring 📋Logging 🔧IaC 💿Backup 🛡️Security

인프라 레이어:VMware ESXi 하이퍼바이저 + vCenter 통합 관리 콘솔 + Virtual SAN 환경 구성으로, 가상화 및 스토리지 통합

오케스트레이션 레이어:중앙관리용 Supervisor 클러스터 및 Workload Cluster 구성 (Control plane 3EA, Worker 9EA)

애플리케이션 레이어:모니터링, 로깅, 백업, 보안을 위한 오픈소스 및 DevOps 도구들을 활용하여 컨테이너 운영 환경 구축

Workflow

요청-승인:컨테이너 관리 포털 연동을 통해 개발환경 제공 (사용자 요청 → 관리자 승인 → 리소스 할당)

개발-배포:Git → Jenkins → Harbor → ArgoCD GitOps 배포

운영-관리:모니터링, 로깅, 백업 및 소스코드, 컨테이너, 클러스터, 런타임 수준의 보안을 고려한 환경 구성

주요 성과

기존 VM 기반 인프라 대비 리소스 효율성 40% 향상
운영 인력 2명으로 140대 이상의 컨테이너 관리, 추후 증설로 200대 이상 관리 구현
연구환경 프로비저닝 시간을 45분→25분으로 단축하여 개발환경 생산성 향상

상황

원인 분석

해결 과정

  • kubectl 명령어를 통한 클러스터 상태 진단
  • Node 리소스 사용량 분석 및 스케줄링 정책 검토
  • Taint/Toleration 재설정 및 리소스 할당 최적화
  • HPA(Horizontal Pod Autoscaler) 설정으로 자동 확장 구성
